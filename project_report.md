# Loan Matching & Credit Decision Platform - Project Report

## 1. Executive Summary
This project implements a comprehensive, research-grade platform for automated loan matchmaking and credit risk assessment. Leveraging real-world data from LendingClub (2007-2018), the system predicts loan approval odds, estimates fair interest rates, and assesses default risk to match borrowers with suitable lenders. The platform is built with a modular Python backend (FastAPI) and an interactive frontend.

## 2. Dataset & Preprocessing
The platform utilizes two primary datasets:
- **Accepted Loans**: Historical data on funded loans, including borrower credit attributes, loan terms, and repayment performance (default/charge-off status).
- **Rejected Loans**: Applications that were declined, providing a crucial negative class for the approval model.

### 2.1 Feature Engineering
To ensure robust model performance, we implemented a sophisticated feature engineering pipeline:
- **Temporal Analysis**: Calculated `months_since_earliest_cr_line` to capture credit history length.
- **Financial Ratios**: Computed `loan_to_income`, `installment_to_income`, and `revol_bal_to_income` to standardize financial health metrics.
- **Transformations**: Applied Log(1+x) transformations to skewed distributions (Annual Income, Revolving Balance) to improve model stability.
- **Categorical Encoding**: One-hot encoding for nominal variables (State, Home Ownership, Purpose).
- **Leakage Prevention**: Strictly separated raw identifiers (e.g., `emp_title`, `url`) from training features, using only engineered numerical (`num__`) and categorical (`cat__`) columns.

## 3. Machine Learning Models
Three distinct XGBoost models were developed to power the decision engine:

### 3.1 Approval Model
- **Goal**: Predict the probability of a loan application being accepted.
- **Data**: Balanced sample of Accepted and Rejected applications.
- **Features**: FICO Score, DTI, Employment Length, Loan Amount, State.
- **Performance**: Achieved high accuracy in distinguishing between clear rejections and funded loans.

### 3.2 Default Risk Model (PD)
- **Goal**: Estimate the Probability of Default (PD) for a given borrower.
- **Data**: Accepted loans with definitive statuses (Fully Paid vs. Charged Off/Default).
- **Methodology**: Treated as a binary classification problem with class imbalance handling (`scale_pos_weight`).
- **Input**: Comprehensive credit profile + *Predicted* Interest Rate (as risk-based pricing implies correlation).

### 3.3 Interest Rate Model
- **Goal**: Predict the fair market interest rate for a loan.
- **Data**: Accepted loans.
- **Methodology**: Regression model minimizing RMSE.
- **Usage**: Provides a baseline "fair rate" which is then adjusted by specific lender risk premiums.

## 4. Lender Matching Algorithm
A rule-based matching engine filters a simulated database of lenders based on hard constraints:
- **Constraints**: Minimum FICO, Maximum DTI, Minimum Income, Allowed States.
- **Dynamic Pricing**: Final offers are generated by applying a lender-specific spread and a risk premium (derived from the Default Risk Model) to the predicted base interest rate.

## 5. System Architecture
- **Backend**: FastAPI (Python) serving REST endpoints for prediction.
- **Inference Engine**: `PredictionService` class that loads serialized models (`.pkl`) and replicates the feature engineering pipeline for real-time data.
- **Frontend**: Lightweight HTML/CSS/JS interface for user interaction.
- **Reproducibility**: `src/` directory contains modular scripts for data pipelining, training, and inference.

## 6. Conclusion & Future Work
The platform successfully demonstrates the viability of automated, data-driven credit decisioning. Future enhancements could include:
- **Deep Learning**: Exploring Neural Networks for capturing complex non-linear interactions.
- **Alternative Data**: Incorporating banking transaction data or alternative credit scores.
- **MLOps**: Implementing model versioning and drift detection pipelines.
